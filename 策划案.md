# 《我们意念合一》策划案

## OurThoughtsAreOne — Game Design Document

---

## 一、模组概述

### 1.1 核心理念

> 还记得你最初接触到《Minecraft》的契机吗？是身边朋友的推荐带你入坑，还是无意刷到的视频勾起了你的兴趣，或是主动搜索某些关键词给你的深入了解？我们通过形形色色的方式与这款游戏结缘，这些点点滴滴的联系指引你我在此相遇，看似不同的道路却最终将我们连接在一起。

**"我们意念合一"** 试图回答一个问题：**如果世界的本质是连接，那么连接本身能不能成为一种力量？**

在 Minecraft 的原生体验中，多人联机时玩家之间虽然有物理碰撞、有物品交换、有视觉上的彼此可见，但本质上每个人依然是一座孤岛。你无法真正"承载"他人的重量，无法建立一种超越聊天框和交易界面的深度联结。

本模组试图用游戏机制重新诠释"联结"——不是简单的 buff 共享，不是冰冷的数值叠加，而是让两个玩家的操作、位移、生存乃至存在本身，产生一种**有机的相互依存**。当你骑乘在同伴肩上替他补充攻击时，当你牵着队友飞越峡谷时，你们不再是两个独立的玩家实体，而是一个**临时的共生体**。

### 1.2 设计目标

- **降低协作门槛**：让玩家之间的协作成为一种直觉行为，而非需要大量沟通的复杂操作
- **创造羁绊体验**：每个核心物品都设计为需要两名玩家共同参与才能发挥最大效用，单人无法激活完整效果
- **风险与收益并存**：联结带来力量，但也意味着责任——你受伤时队友替你分担，同时也意味着队友受伤时会拖累你
- **轻量但深刻**：不引入复杂的 GUI、技能树或数值系统，仅通过少数几个物品改变玩家间的交互范式

### 1.3 模组信息

| 项目 | 内容 |
|------|------|
| 模组名称 | 我们意念合一（OurThoughtsAreOne） |
| 适用版本 | Minecraft 1.26.1 (NeoForge) |
| 模组类型 | 多人协作 / 机制扩展 |
| 核心玩法 | 玩家间的深度联结与协同操作 |
| 目标玩家 | 双人/小团队生存、冒险地图、解谜地图 |

## 二、世界观叙事（可选调味）

在世界的底层代码中，每一个实体都是一串孤独的数据流。但某些古老的算法证明：当两个灵魂的"频率"足够接近时，它们可以通过特殊的媒介——**核心**——短暂地共享同一个存在坐标。

- **投射核心** 是"信任"的具象化：你将自己的一部分存在"投射"到另一个人的位置上，你们的攻击节奏同步，你们的伤害共享。这是一种**单向的托付**。
- **飞行核心** 是"带领"的具象化：你向另一个灵魂伸出手，对方选择握住。这是一种**双向的选择**——邀请可以被拒绝，连接可以被断开。

这些核心并非武器或工具，它们是**关系的容器**。

## 三、核心物品设计

### 3.1 物品 1：投射核心（Projection Core）

#### 3.1.1 设计意图

投射核心解决的痛点是：在 Minecraft 的多人探索中，总有一个玩家负责战斗、一个玩家负责建造/挖掘。但战斗玩家冲在前面时，建造玩家常常被落在后面；而建造玩家在高空作业时，战斗玩家只能在下面干看着。投射核心让两个玩家**共享同一个物理位置**，同时**保留各自的操作能力**——一个负责移动和攻击，一个负责交互和建造。

#### 3.1.2 物品外观

- 一个深蓝色的半透明立方体，内部有流动的光点纹理，像一颗被压缩的星空
- 手持时发出微弱的蓝色粒子效果
- 合成配方：4×末影珍珠 + 1×下界之星 + 2×紫水晶碎片 + 2×铁锭

#### 3.1.3 核心机制

**阶段一：投射（发起联结）**

| 操作 | 效果 |
|------|------|
| 玩家 A 手持投射核心，右键玩家 B | A 乘骑到 B 身上（视觉上 A 变为半透明状态悬浮在 B 的上方偏后位置） |
| 联结建立 | 双方屏幕边缘出现微弱的蓝色光晕，提示联结状态已激活 |
| 联结建立后 | 投射核心进入冷却状态（30 秒），冷却期间无法再次投射/切换投射目标 |

**阶段二：共生（联结中的行为规则）**

| 角色 | 能力 | 限制 |
|------|------|------|
| **乘骑者 A** | 可以放置方块、破坏方块、使用物品、与方块交互（打开箱子、拉杆等） | 无法攻击任何实体；无法主动移动（移动完全依附于被乘骑者 B） |
| **被乘骑者 B** | 正常移动、跳跃、飞行（如已有飞行能力）；正常攻击 | 无额外限制 |

**阶段三：协同攻击（被动触发）**

- 当被乘骑者 B 攻击任意实体时，乘骑者 A 自动以相同攻击力进行一次**追加攻击**（A 不消耗耐久，不触发攻击冷却，延迟 2 tick 内判定）
- 追加攻击的伤害 = A 当前主手武器伤害 × 0.8
- 追加攻击带有独特的蓝色粒子轨迹，区别于普通攻击
- 追加攻击无视 A 的攻击冷却（但 A 主动攻击仍然受冷却限制）

**阶段四：伤害分摊**

| 条件 | 效果 |
|------|------|
| 被乘骑者 B 受到伤害 | 伤害的 25% 由乘骑者 A 承担（A 的护甲值参与减免），B 承受 75% |
| 乘骑者 A 受到伤害（环境伤害、魔法伤害等） | 伤害的 25% 由被乘骑者 B 承担，A 承受 75% |
| B 获得额外护甲 | 乘骑时，被乘骑者 B 获得乘骑者 A 当前护甲值的 25% 作为额外护甲（以盔甲架的形式显示在 B 周围，持续闪烁蓝色粒子） |

**阶段五：解除联结**

| 触发条件 | 效果 |
|------|------|
| 乘骑者 A 按下潜行键（Shift） | 从 B 身上下来，联结解除，双方恢复正常状态 |
| 任意一方死亡 | 联结解除，存活方获得 3 秒的"共鸣残留"效果（速度 I + 抗性提升 I） |
| 双方距离超过 32 格 | 联结强制解除，投射核心进入 60 秒超长冷却 |
| A 切换投射核心到其他物品栏位 | 联结解除（但投射核心仍在主手时不会解除） |

#### 3.1.4 视觉与音效设计

| 时机 | 视觉 | 音效 |
|------|------|------|
| 联结建立 | A 化为蓝色光粒汇聚到 B 身上，然后显形为半透明悬浮状态 | 低沉的共鸣音（类似末影人传送但更柔和） |
| 追加攻击触发 | 蓝色的剑影从 A 的位置射出，击中 B 正在攻击的目标 | 短促的玻璃碎裂音 + 微弱的钟声 |
| 伤害分摊 | 受到分摊伤害的一方身上闪过蓝色裂纹纹理 | 心跳声（仅受分摊伤害方听到） |
| 联结解除 | 光粒从 B 身上散开，A 在原位置后方实体化 | 共鸣音的反向播放（渐弱） |

#### 3.1.5 特殊交互

- **骑乘叠加**：如果一个投射核心的乘骑者自身被另一个投射核心乘骑，形成"A 骑 B，C 骑 A"的三人链。追加攻击沿链向上传递（C→A→B，每次传递伤害衰减 20%），伤害分摊也沿链向上传递（每层额外衰减 10%）。链条最长 4 人。
- **鞘翅兼容**：被乘骑者 B 使用鞘翅滑翔时，A 也会跟随滑翔（视觉上 A 保持悬浮在 B 身后的相对位置）。
- **游泳兼容**：被乘骑者 B 在水中游泳时，A 获得水下呼吸效果（因为 A 不消耗氧气值）。

### 3.2 物品 2：飞行核心（Flight Core）

#### 3.2.1 设计意图

飞行核心解决的痛点是：Minecraft 中的队伍移动效率极低，尤其在跨越复杂地形时，总有人掉队。鞘翅需要烟花火箭且门槛高，骑马被地形限制，矿车需要铺设铁轨。飞行核心提供了一种**基于信任的快速双人移动方案**——一个人飞，一个人被带飞。

#### 3.2.2 物品外观

- 一个淡金色的半透明球体，内部有旋转的羽毛状纹理
- 手持时散发金色粒子向上飘升
- 副手装备时，玩家背后出现一对半透明的光翼（非实体，纯视觉效果）
- 合成配方：2×幻翼膜 + 1×下界之星 + 3×金锭 + 2×羽毛 + 1×末影珍珠

#### 3.2.3 核心机制

**阶段一：飞行状态（副手装备）**

| 条件 | 效果 |
|------|------|
| 玩家 A 将飞行核心放置在副手 | A 进入**常态飞行状态**（类似创造模式飞行，但速度为创造飞行的 60%） |
| 飞行操控 | 跳跃键上升，潜行键下降，WASD 控制水平移动 |
| 飞行耐力 | A 拥有飞行耐力条（显示在经验条上方），初始 100 点 |
| 耐力消耗 | 飞行时每秒消耗 2 点；连接队友后每秒额外消耗 1 点/每名被连接者 |
| 耐力恢复 | 落地（脚触方块）后每秒恢复 5 点 |
| 耐力耗尽 | 耐力归零时强制落地，A 获得 5 秒缓慢 II 效果，10 秒内耐力不恢复 |
| 副手切换 | 将飞行核心从副手移出，立即结束飞行状态（A 安全落地，被连接者由下文规则处理） |

**阶段二：链接邀请（主动发起）**

| 操作 | 效果 |
|------|------|
| 飞行中的 A 右键准星指向的 16 格内玩家 B | 向 B 发送**链接邀请** |
| B 的屏幕显示提示 | 屏幕中央出现半透明的金色提示文字："[玩家 A] 向你伸出链接之手 — [左键 接受] [右键 拒绝]" |
| 邀请超时 | 8 秒内 B 未作出选择，邀请自动过期 |
| 邀请冷却 | A 向同一玩家发送邀请有 15 秒冷却（防止骚扰） |

**阶段三：链接建立（B 接受邀请后）**

| 效果 | 详情 |
|------|------|
| 视觉连接 | A 和 B 之间出现一条半透明的金色光链（贝塞尔曲线，轻微摆动），链条材质为流动的光粒 |
| 最大链长 | 初始 16 格，超出时产生拉力将 B 拉向 A |
| 拖拉力度 | B 被以 A 飞行速度的 80% 拉向 A 的方向，拉力平滑加速，非瞬间传送 |
| B 的状态 | B 被悬挂在链条末端，双脚离地，处于"被牵引"状态；B 可以小幅度摆动（左右偏移最大 3 格），但无法主动改变主要移动方向 |
| B 的操作 | B 无法攻击、无法放置/破坏方块，但可以旋转视角、使用聊天、打开背包 |
| A 的操作 | 飞行操控不变，但飞行耐力消耗增加（见阶段一）。A 无法攻击 |

**阶段四：拒绝邀请**

- B 按下右键拒绝后，A 的屏幕提示："[玩家 B] 婉拒了你的链接邀请"
- A 的飞行核心进入 5 秒冷却（冷却期间无法发送新的邀请）
- B 不会受到任何负面效果

**阶段五：断开链接**

| 触发条件 | 效果 |
|------|------|
| A 按下潜行键（Shift） | 链接断开，B 获得 3 秒缓降效果后安全落地 |
| B 按下潜行键（Shift） | B 主动挣脱链接，获得 3 秒缓降效果后安全落地 |
| A 将飞行核心从副手移出 | 链接断开，B 获得缓降效果 |
| A 飞行耐力耗尽 | 链接断开，A 和 B 均获得缓降效果 |
| A 和 B 之间出现固体方块阻隔 | 链接闪烁 3 秒后断开（给玩家时间清理路径），B 获得缓降效果 |
| 任意一方死亡 | 链接断开，飞行核心进入 60 秒冷却 |
| A 与 B 距离超过 32 格（拉力无法拉回时） | 链接强制断开 |

**阶段六：多链接**

- 飞行核心可以同时链接多名玩家，每多链接一名玩家，A 的耐力消耗 +1 点/秒
- 所有被链接者共享同一条链条的视觉（多条链从 A 出发，呈扇形展开）
- 最大链接数：4 人
- 被链接的玩家之间不会互相碰撞

#### 3.2.4 飞行核心与投射核心的联动

两种核心可以同时使用：

| 场景 | 效果 |
|------|------|
| 玩家 A 副手装备飞行核心（正在飞行） | A 处于飞行状态 |
| 玩家 B 使用投射核心乘骑在 A 身上 | B 成为乘骑者，可以进行建造/交互 |
| A 再用飞行核心链接玩家 C | C 被光链牵引飞行 |
| 形成"A（飞行员）← B（乘骑建造者）+ C（被牵引者）" | 三人联结协作：一人驾驶，一人建造，一人待命 |

这是本模组的最终协作形态——**三位一体协作链**。

#### 3.2.5 视觉与音效设计

| 时机 | 视觉 | 音效 |
|------|------|------|
| 副手装备飞行核心 | 玩家背后展开半透明光翼，脚下出现轻微的金色光尘 | 轻柔的风铃声 |
| 发送链接邀请 | 从 A 手中射出一道金色光粒飞向 B | 竖琴上行琶音 |
| B 接受邀请 | 光粒在 B 身上绽开，光链瞬间生成并连接双方 | 竖琴下行琶音 + 低音弦乐持续 |
| B 拒绝邀请 | 光粒在 B 身上消散，金色变为灰色飘落 | 软弱的单音（类似音符盒低音） |
| 链接断开 | 光链从中间断裂、两端分别向 A 和 B 收缩消失 | 弦乐渐弱 |
| 耐力耗尽 | 光翼急速闪烁红色后消失 | 刺耳的玻璃碎裂音 |

## 四、辅助物品设计

### 4.1 共鸣石（Resonance Stone）

#### 4.1.1 设计意图

在大型服务器中，你如何找到那个愿意与你建立联结的人？共鸣石提供了一种非语言的"寻找同伴"的方式——它不显示坐标，不破坏沉浸感，但能让你感知到附近同样渴望联结的灵魂。

#### 4.1.2 物品外观

- 两块可以配对的深紫色圆石，表面有微弱的脉动光纹
- 合成配方：1×末影珍珠 + 4×紫水晶碎片 + 4×深板岩圆石（合成产出 2 个绑定的共鸣石）

#### 4.1.3 机制

- 合成时同时产出两块**已绑定的共鸣石**（堆叠中显示为"共鸣石 [已绑定]"）
- 将一块共鸣石交给另一名玩家后：
  - 双方手持共鸣石时，另一名玩家的位置会以**屏幕边缘的淡紫色光晕方向**被感知（类似声纳，越靠近光晕越亮）
  - 双方距离超过 64 格时，共鸣石开始发热（粒子效果增强），指向效果加强
  - 双方距离在 16 格内时，共鸣石发出轻微的共鸣音
- 其中一方死亡时，另一方手中的共鸣石会碎裂（消失并发出破碎音效），象征联结的终结
- 一块共鸣石只能绑定一对玩家；如果要更换绑定对象，需要在铁砧上用另一块未绑定的共鸣石覆盖

### 4.2 思念信标（Beacon of Longing）

#### 4.2.1 设计意图

Minecraft 原版信标是向下的——它为你提供 buff，但只在一个固定范围内有效。思念信标是"向上"的——它让你向远方的人发出一个可见的信号，告诉对方"我在这里"。

#### 4.2.2 物品外观

- 放置在方块上的小型信标，仅 1 格高，散发淡蓝色光柱（比原版信标光柱更细、更柔和）
- 合成配方：1×原版信标 + 1×投射核心 + 1×共鸣石

#### 4.2.3 机制

- 放置后，思念信标向天空射出一道细的淡蓝色光柱，在任意距离可见（但仅在其所在维度的玩家可见）
- 手持共鸣石的玩家可以在 512 格内看到思念信标的光柱穿墙提示（光柱轮廓线可见）
- 思念信标的主人（放置者）死亡时，光柱变为红色持续 30 秒，提醒同伴
- 最多同时存在 3 个思念信标（每个玩家），超过时最早的自动拆除掉落

## 五、平衡性考量

### 5.1 投射核心

| 维度 | 分析 |
|------|------|
| **PvE 强度** | 追加攻击相当于 1.8 倍单人输出，但需要两人聚在一起，AOE 伤害下风险翻倍。属于高风险高回报。 |
| **PvP 强度** | 伤害分摊机制意味着集火被乘骑者 B 同时也在消耗 A。在 PvP 中，联结是一把双刃剑。 |
| **建造效率** | 显著提升——一人负责走位，一人负责建造，在高空作业或复杂地形中优势明显。 |
| **反制手段** | 被乘骑者可以随时 Shift 将乘骑者甩下；乘骑者无法攻击意味着纯粹的战斗玩家不会选择骑乘。 |

### 5.2 飞行核心

| 维度 | 分析 |
|------|------|
| **探索效率** | 大幅提升队伍移动速度，但耐力系统限制了无限飞行。 |
| **PvP 影响** | 飞行核心在 PvP 中禁用（被攻击时飞行中断 3 秒，且无法在战斗状态下发送链接邀请）。 |
| **建造配合** | 与投射核心联动后，可实现"空运建造队"——飞行员 + 建造者 + 物资运输者。 |
| **反制手段** | 弓箭可以打断飞行（被弹射物击中，耐力额外扣 10 点）；被链接者可以随时挣脱。 |

### 5.3 通用限制

- 所有核心物品在下界和末地中**效果衰减**：投射核心追加攻击伤害 ×0.7，飞行核心飞行速度 ×0.7，耐力消耗 ×1.5
- 核心物品不可堆叠，不可放入潜影盒
- 玩家不能同时成为两个投射核心的"被乘骑者"（防止叠加混乱）

## 八、主题深化：关于"联结"的再思考

### 8.1 为什么叫"我们意念合一"

这个模组名称有两个层面的含义：

**字面层**：投射核心让两个人的操作协同如同一个人——你移动，我建造；你攻击，我补充。飞行核心让两个人的位移合而为一——我飞，你随。这种操作层面的无缝协作，像是两个人的意念暂时合并在了一起。

**隐喻层**：在 Minecraft 的多人世界里，联结不是理所当然的。你需要主动伸出手（手持核心右键），需要对方的同意（接受邀请），需要承担风险（伤害分摊），也需要付出代价（耐力消耗）。**真正的联结从来不是免费的力量**。模组希望玩家在享受协作便利的同时，也感受到"被联结"的重量——当你受伤时，你的队友也在疼；当你飞得太高时，你手里牵着的人也在空中。

### 8.2 主题物的刻意对称

| 投射核心 | 飞行核心 |
|------|------|
| 联结是**垂直的**（骑乘/承载） | 联结是**水平的**（牵引/同行） |
| 联结是**单向的托付**（我把自己交给你） | 联结是**双向的选择**（你愿意跟我走吗？） |
| 力量来自**共享伤害** | 力量来自**共享位移** |
| 代表了"**承担**" | 代表了"**带领**" |
| 暗合"**我与你同在**" | 暗合"**跟我来**" |

两种核心既是独立的协作工具，又可以在同一时刻叠加使用，形成完整的协作体验闭环。
